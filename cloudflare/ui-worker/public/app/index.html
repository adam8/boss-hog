price_only uses only price-derived features. core_fundamentals adds the same-report hog fundamentals.

In simple terms, this setting decides whether the model looks only at price behavior or also at the physical hog-market details in the same USDA report. The deeper pack gives RBP more context for deciding which past months were truly similar.

How many recent monthly observations to keep before building the supervised dataset.

This is the amount of history you let the backtest use. A larger number gives the model more past months to learn from, while a smaller number focuses it on the more recent market regime.

How many supervised rows are used before the first out-of-sample prediction.

Think of this as the minimum amount of history the model needs before it starts making realistic test predictions. A bigger starting window usually means the early predictions are based on a fuller training sample.

How many additional sparse-grid cells RBP samples in each fit.

RBP tries many small views of the data rather than one fixed equation. More random cells means it explores more combinations of variables and thresholds, which can help it catch structure but also takes longer to run.

The sparse-grid sampler seed. Keep it fixed when you want comparable runs.

The seed keeps the randomized parts of the RBP search repeatable. If you rerun with the same seed and same settings, you should get the same result instead of a slightly different sample of model cells.
